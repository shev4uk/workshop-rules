# Open PRD Document

Opens the Product Requirements Document for reference.

## Steps

1. Open `docs/PRD.md` in the editor
2. Review project goals, scope, and acceptance criteria

## Usage

Run this command when you want to:
- Check project scope and requirements
- Verify changes align with PRD goals
- Review acceptance criteria before marking tasks complete

## Notes

- PRD contains workshop structure, learning goals, and success metrics
- Use it to ensure slide content matches workshop objectives
- Update PRD if scope changes significantly
